# OpenClash 助手 PRD

## 1. 产品定位

OpenClash 助手不是高级玩家工具箱，而是面向小白用户、售后和装机商家的“助手式产品”。

核心体验：

1. 一键配置
2. 一键检测
3. 一键建议
4. 说人话
5. 检测结果 -> 修复建议 -> 一键动作

## 2. 目标用户

1. 不会写 YAML、不懂规则组、不懂 DNS 的普通用户
2. 卖软路由、做售后、批量装机的商家
3. 想快速完成订阅导入、配置生成、故障检测的人
4. 需要快速确认流媒体、AI 服务、DNS 是否正常的人

## 3. 核心使用场景

### 3.1 首次安装

1. 导入订阅
2. 选择用途
3. 生成配置
4. 跑一键体检
5. 应用建议
6. 启用 OpenClash

### 3.2 网络排障

1. 一键体检
2. 定位 DNS / 节点 / 分流问题
3. 给出修复动作
4. 一键执行
5. 自动复检

### 3.3 节点优化

1. 跑网站走向检测
2. 跑流媒体测试
3. 跑 AI 服务测试
4. 生成自动切换和节点用途建议
5. 用户确认应用

## 5. MVP 范围

一期必须完成：

1. 本地订阅转换前后端接入
2. 一键导入订阅
3. 一键转换配置
4. 一键推荐配置
5. 场景配置 + 运行概览
6. 一键体检
7. 网站走向检测
8. 流媒体测试
9. AI 服务测试
10. DNS 检测与刷新
11. 自动切换建议

## 6. 页面原则

1. 首页不是按钮墙，而是任务入口 + 状态概览 + 建议摘要
2. 第二页必须是“基础场景配置 + 运行概览 + 基础建议”
3. 所有专项检测都要先给人话结论，再给底层明细
4. 默认小白模式隐藏过多底层术语
5. 高级模式才展示链路、HTTP、命令和原始字段

## 7. 关键模块定义

### 7.1 一键导入订阅

目标：
把订阅链接或配置文件变成可用输入源，并先判断是否适合直接导入。

输出：

1. 是否读取成功
2. 是否建议转换
3. 是否已有策略组
4. 节点数量
5. 下一步建议

### 7.2 一键转换配置

模板：

1. 小白默认版
2. 流媒体版
3. AI 工具版
4. 游戏低延迟版
5. 稳定优先版
6. 轻量省内存版

输出：

1. 当前模板
2. 推荐模板
3. 转换后端
4. 写入 OpenClash 的地址
5. 一键写入入口

### 7.3 一键体检

体检必须覆盖：

1. 基础服务
2. 订阅与配置
3. DNS 状态
4. 流媒体 / AI 连通
5. 网站走向
6. 自动切换建议

每项必须包含：

1. 状态等级
2. 人话解释
3. 下一步建议
4. 一键动作

### 7.4 网站走向检测

定义：
让用户知道网站最后走了哪条路、由哪个出口出去。

页面要点：

1. 小白模式优先展示“走向是否合理”
2. 高级模式再展示出口、HTTP、延迟
3. 支持把结果联动到配置建议

### 7.5 流媒体检测

目标：
不只看能不能访问，还要判断是否适合作为长期节点。

输出：

1. 可访问性
2. 地区
3. 稳定性
4. 长期建议

### 7.6 AI 服务测试

目标：
判断 ChatGPT、Claude、Gemini 等是否适合当前线路。

输出：

1. 连通性
2. 延迟
3. 线路适配性
4. 是否建议切换节点

### 7.7 DNS 检测与修复

目标：
优先解决“网站打不开、时好时坏”的真实问题。

输出：

1. 解析链路状态
2. 诊断摘要
3. 建议动作
4. 一键刷新

## 8. 交互要求

### 8.1 小白模式

1. 不直接展示 Fake-IP、策略组命中等术语
2. 优先展示结论、建议、动作
3. 默认展开体检和建议

### 8.2 高级模式

1. 显示更多状态字段
2. 显示 UCI 命令和地址
3. 显示更完整的检测明细
